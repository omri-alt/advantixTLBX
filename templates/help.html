Kelkoo

Late sales postbacks

Compares the two latest SalesReport_7days-generated-* tabs on the KLtools workbook. Sale dates must fall in the late-sale window (excludes on-time “yesterday” sales vs each tab’s generation date). Skips rows if click_id already appears in a daily SalesReport_*_generated-* tab or if {month}_late_sales_log already recorded a LateSale postback.

FieldDescription
As-of dateOptional YYYY-MM-DD; blank = latest tab.
ModeDry run shows GET URLs only. Apply sends GETs and appends successes to the monthly log.

Example: leave as-of blank, choose Dry run, submit to preview.

Daily conversion postbacks

Per-feed pages run Kelkoo / Adexa / Yadore daily click & sale GET postbacks (see integrations/daily_conversion_postbacks.py). State file: runtime/daily_conversion_postbacks_state.json (or DAILY_CONVERSION_POSTBACK_STATE_PATH). On a feed page, set Report date to inspect another day’s resume rows or use ?date=YYYY-MM-DD in the URL.

Run form: Ignore saved progress maps to --no-resume. Clear resume state wipes saved progress for this feed + report date before the run (use when you intentionally want a clean rerun).

Blend

Blend workflows (summary)

Blend workflow runs run_blend_workflow.py: Keitaro sync from the Blend sheet, refresh potential sheets, optional populate. Blend sheet → Keitaro runs only blend_sync_from_sheet.py (prune bad auto=v, rebuild offers/flows).

Blend Stop Closed (blend_stop_closed_merchants.py): during-day, sets non-monetized auto=v Kelkoo1/2 Blend rows share to 0.

Env: BLEND_SHEETS_SPREADSHEET_ID, BLEND_POTENTIAL_FEEDS, feed API keys. Full URL rules are documented in CLAUDE.md (Blend → Keitaro).

Console pages share nav: Sheet lists, Campaign WL, Sync BL, WL check, Action items. State and last-run metadata live in runtime/ecomnia_console_state.json. Sheet ID: EC_SHEETS_SPREADSHEET_ID; geo tab name ECOMNIA_GLOBA_LIST_TAB (default globaList).

Sheet lists: load globaList → per-geo blacklist/whitelist copy blocks.

Campaign whitelist: API whitelistsources + merged sheet view.

Sync blacklists: dry-run vs apply; skips -wl campaigns.

Action items: under-spend + zero-click sources; CPC bumps remain manual in trackExploration.

Campaign whitelist (detail)

Each campaign exposes whitelistsources. Global whitelist = source on ≥2 campaigns. Checks merge this with sheet geo whitelist. Potential (0-click) = on merged WL but no clicks in the stats window — drill in to set cpcbysource.

Sync blacklists (detail)

Same idea as legacy explorations_blacklist_synch: sources blacklisted in 4+ campaigns in a geo are added to other campaigns in that geo. Skips -wl campaigns. Use Dry-run then Apply.

Open Sync BL →

Whitelist traffic check (detail)

Each row: campaign × source on the merged whitelist (campaign whitelistsources ∪ sheet geo WL). Clicks from Ecomnia in the window.

Admin — single source audit: lists campaigns where the source is on the merged whitelist, has clicks in adv-stats-by-source for the window, and reported conversions/sales are 0. Default minimum 2 campaigns filters cross-campaign patterns; set Min. campaigns to 1 to see every match. If the API omits conversion columns, 0 conversions is assumed.

Open WL check →

Action items (detail)

Active non--wl campaigns where yesterday spend < daily budget and the by-source report shows zero-click sources (not blacklisted). Approve CPC bumps in sheet trackExploration column CpcLvlUp (v/x); this UI does not write bids.

AutoServer automations

Scheduler & manual triggers

APScheduler fires an hourly signal at minute 0. Jobs: Mehilot, KLFIX optimize, Pause unmon SK, KLWL (even hours), QualityWL (even hours), Close Nipuhim (hour 23). Env aliases are synced from config via integrations/autoserver/env.py.

APIPurpose
GET /api/automationsStatus + last run per job.
GET /api/automations/log?limit=20Recent run log entries.
GET /api/automations/trigger/all202 — queue all run_manually.
GET /api/automations/trigger/<ClassName>202 — queue one job.

Run log file: data/autoserver_run_log.json (trimmed to AUTOSERVER_RUN_LOG_MAX). Disable scheduler on extra Gunicorn workers: AUTOSERVER_SCHEDULER_ENABLED=0.
